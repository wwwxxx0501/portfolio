1   项目背景

本项目设计并实现了一套基于ESP32-S3和ROS2的完整智能扫地机器人系统，采用上下位机分离架构：ESP32-S3作为下位机负责底层传感器采集与电机控制，ROS2作为上位机负责SLAM建图、AMCL定位、Nav2路径规划与自主导航。两者通过自定义轻量级WiFi UDP协议通信，替代资源占用较大的micro-ROS方案，在保持ROS2导航栈完整兼容的前提下大幅降低嵌入式端资源开销。

3   下位机固件——FreeRTOS多任务架构

ESP32-S3固件基于FreeRTOS构建，核心任务包括：

  • 电机控制任务：PCNT硬件编码器计数采集轮速，增量式PID闭环控制（50Hz采样），双路RZ7899 PWM驱动
  • 激光雷达任务：CoinD6串口数据采集（UART 230400bps），原始扫描数据打包转发至上位机
  • IMU采集任务：MPU6500 I2C数据读取与发布
  • UDP通信任务：WiFi连接管理，线程安全的数据帧收发
  • 安全监控任务：电机堵转检测、过流保护、异常制动

4   PID闭环速度控制

增量式PID速度控制器（50Hz采样周期）
typedef struct {
    float Kp, Ki, Kd;       // Kp=0.1, Ki=0.1, Kd=0.001
    float error, last_error, prev_error;
    float output;
    float out_max, out_min;
} PID_TypeDef;

float PID_Incremental(PID_TypeDef *pid, float target, float actual) {
    pid->error = target - actual;

    // 增量式PID: delta_u = Kp*(e-e1) + Ki*e + Kd*(e-2e1+e2)
    float delta = pid->Kp * (pid->error - pid->last_error)
                + pid->Ki * pid->error
                + pid->Kd * (pid->error - 2*pid->last_error
                             + pid->prev_error);

    pid->output += delta;

    // 输出限幅，防止PWM溢出
    if (pid->output > pid->out_max) pid->output = pid->out_max;
    if (pid->output < pid->out_min) pid->output = pid->out_min;

    pid->prev_error = pid->last_error;
    pid->last_error = pid->error;

    return pid->output;
}

5   IIR低通滤波器——编码器速度平滑

编码器原始速度信号含有高频噪声，采用一阶IIR低通滤波器（截止频率15Hz）进行平滑处理后再送入PID控制器：

H(z) = (a)/(1-(1-a)z-1), a = (Δ t)/((1)/(2π fc) + Δ t)
IIR低通滤波器实现（截止频率15Hz）
void IIR_LPF_Init(IIR_LPF* lpf, float cutoff_hz, float dt_s) {
    float RC = 1.0f / (2.0f * M_PI * cutoff_hz);
    lpf->a = dt_s / (RC + dt_s);
    lpf->y = 0.0f;
}

float IIR_LPF_Update(IIR_LPF* lpf, float x) {
    lpf->y += lpf->a * (x - lpf->y);
    return lpf->y;
}

6   电机安全监控

下位机通过ADC实时监测外部电压信号，实现电机堵转和过流保护。检测逻辑采用连续计数消抖机制：每50ms采样一次ADC，若电压连续5次超过2.3V阈值则禁用电机并将速度归零；连续5次低于阈值则恢复正常控制。

#define CONSECUTIVE_COUNT   5     // 连续检测次数阈值
#define CHECK_PERIOD_MS     50    // 检测周期(ms)

7   WiFi UDP通信协议

采用自定义轻量级UDP协议替代micro-ROS，定义5种帧类型：

方向帧头内容频率校验
上行（ESP32→PC）0xFD 0xEA左右轮速度+时间戳50HzXOR
上行（ESP32→PC）0xEB 0xBECoinD6激光雷达原始数据10HzXOR
上行（ESP32→PC）0xEF 0xFEESP32本机IP广播1HzXOR
下行（PC→ESP32）0xEE 0xFF线速度+角速度指令50HzXOR
下行（PC→ESP32）0xEA 0xAE清扫电机控制指令按需XOR

NTP风格时间同步：设计时间同步请求帧（0xFC 0xCF，10字节）和应答帧（27字节，含T1/T2/T3三个时间戳），借鉴NTP协议实现ESP32与PC间毫秒级时钟同步。时间偏移计算公式：

offset = ((T1 - T2) + (T4 - T3))/(2)

8   ROS2上位机架构

上位机基于ROS2 Jazzy/Humble构建完整导航栈，包含9个自研功能包：

  • UDP桥接节点（speedy\udp\bridge）：接收ESP32上行数据，转换为ROS2标准话题（/odom, /scan），发布里程计与激光雷达数据
  • Cartographer SLAM建图：融合激光雷达扫描数据与轮式里程计，实时构建2D栅格地图
  • AMCL粒子滤波定位：在已知地图中实现机器人精确定位（<5cm精度），粒子数500~3000自适应
  • Nav2导航栈：全局规划（NavFn）+ 局部规划（MPPI控制器，20Hz控制频率），实现自主路径规划与避障
  • Frontier-Based自主探索：自动检测地图边界未探索区域，驱动机器人完成全屋建图
  • Boustrophedon覆盖路径规划：Zig-Zag弓字形全覆盖清扫，上位机统计覆盖率
  • Web控制界面：FastAPI + Vue.js + WebSocket实时通信，提供远程遥控、建图模式切换、状态监控，无需安装客户端

8.1   UDP桥接节点——里程计发布

UDP桥接节点接收ESP32上行电机速度帧后，进行差速运动学逆解与位姿积分，发布标准ROS2 Odometry消息：

ROS2 UDP桥接节点里程计发布（C++）
void update_odometry(int64_t time, float left_speed,
                     float right_speed) {
    // 时间同步补偿
    int64_t current_time = time * 1000
                         + PacketProcessorBase::time_offset_ns;
    double dt = (current_time - last_time_) / 1e9;
    last_time_ = current_time;

    // 差速运动学逆解
    double linear_vel  = (left_speed + right_speed) / 2.0;
    double angular_vel = (right_speed - left_speed) / wheel_base_;

    // 位姿积分
    theta_ += angular_vel * dt * theta_tuning_;
    x_ += linear_vel * dt * cos(theta_);
    y_ += linear_vel * dt * sin(theta_);

    // 发布ROS2 Odometry消息
    nav_msgs::msg::Odometry odom;
    odom.header.stamp = rclcpp::Time(current_time);
    odom.header.frame_id = "odom";
    odom.child_frame_id = "base_footprint";
    odom.pose.pose.position.x = x_;
    odom.pose.pose.position.y = y_;
    odom.pose.pose.orientation = quaternion_from_yaw(theta_);
    pub_->publish(odom);
}

其中 wheel\base\_ = 0.18m（轮距），theta\tuning\_ = 0.97515528（航向角校准系数，通过实际标定获得）。

9   差速运动学与里程计

采用差速驱动运动学模型，PCNT硬件编码器采集轮速后进行里程计积分：

Δ s = (Δ sL + Δ sR)/(2), Δθ = (Δ sR - Δ sL)/(D)
xk+1 = xk + Δ s · cosθk + (Δθ)/(2), yk+1 = yk + Δ s · sinθk + (Δθ)/(2)

其中 D 为两轮轴距，Δ sL、Δ sR 分别为左右轮弧长增量。里程计数据通过UDP桥接节点发布为ROS2标准Odometry消息，供Cartographer和AMCL融合使用。
