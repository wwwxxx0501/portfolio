SpeedyBot 自主建图与导航机器人

薛雅文 2025

SpeedyBot 是一款面向室内场景的自主建图与导航小型机器人。项目采用上下位机分层架构： 下位机基于 ESP32-S3 实现电机 PID 闭环控制、编码器读取、IMU 数据融合与 WiFi UDP 实时通信； 上位机运行 ROS2 框架，集成 Cartographer 激光 SLAM 建图、AMCL 自适应蒙特卡洛定位 与 Nav2 全局/局部路径规划。硬件端设计并制作了包含电源管理、RZ7899 电机驱动、 九轴 IMU（CoinD6）以及 ESD 防护在内的定制 PCB。

关键词：ESP32-S3, ROS2, Cartographer SLAM, Nav2, PID, 自主导航, 定制 PCB

1   系统架构

系统分为上位机（PC / Raspberry Pi + ROS2）与下位机（ESP32-S3）两层。 两层之间通过 WiFi UDP 协议实时通信，上位机下发速度指令（cmd_vel）， 下位机上报里程计（odom）与 IMU 数据。分层设计使算法开发与底层驱动解耦， 便于独立迭代。

层级硬件职责
上位机PC / RPi + LiDARSLAM 建图、定位、路径规划、决策
下位机ESP32-S3 + 定制 PCB电机驱动、编码器、IMU、UDP 通信

2   下位机：ESP32-S3 固件

2.1   电机 PID 控制

双路直流减速电机通过 RZ7899 H 桥驱动芯片控制。采用增量式 PID 控制器闭环调节转速：

Δu[k] = Kp(e[k] − e[k−1]) + Ki·e[k] + Kd(e[k] − 2e[k−1] + e[k−2])
motor_pid.c — 增量式 PID 控制核心
typedef struct {
    float Kp, Ki, Kd;
    float err, err_last, err_prev;
    float output;
} PID_t;

void PID_Update(PID_t *pid, float target, float actual) {
    pid->err = target - actual;
    float delta = pid->Kp * (pid->err - pid->err_last)
                + pid->Ki * pid->err
                + pid->Kd * (pid->err - 2*pid->err_last + pid->err_prev);
    pid->output += delta;
    // 输出限幅
    if (pid->output > PWM_MAX)  pid->output = PWM_MAX;
    if (pid->output < -PWM_MAX) pid->output = -PWM_MAX;
    pid->err_prev = pid->err_last;
    pid->err_last = pid->err;
}

编码器通过 PCNT（Pulse Counter）外设读取脉冲，计算实际转速并反馈至 PID。 控制周期 20 ms（50 Hz），与 ROS2 的 odom 发布频率一致。

2.2   IMU 数据融合

板载 CoinD6 九轴 IMU（三轴加速度 + 三轴陀螺仪 + 三轴磁力计）， 通过 I²C 读取原始数据，经卡尔曼滤波融合后输出四元数姿态。 IMU 数据随 odom 一并通过 UDP 上报，用于 SLAM 的运动预测环节。

2.3   WiFi UDP 实时通信

ESP32-S3 的 WiFi Station 模式连接至与上位机相同的局域网。 通信协议采用自定义二进制帧格式：帧头（2B）+ 类型（1B）+ 数据长度（2B）+ 数据 + CRC16 校验。 上位机以 50 Hz 下发 cmd_vel（线速度 v + 角速度 ω）， 下位机以 50 Hz 上报 odom（位移 dx, dy, dθ）+ IMU（四元数 + 角速度）。

udp_comm.c — 自定义二进制帧收发
#define FRAME_HEAD  0xAA55
#define CMD_VEL     0x01
#define ODOM_IMU    0x02

typedef struct __attribute__((packed)) {
    uint16_t head;
    uint8_t  type;
    uint16_t len;
    uint8_t  data[];
} Frame_t;

/* 发送 odom + IMU 帧 */
void UDP_SendOdom(int sock, struct sockaddr_in *addr,
                  float dx, float dy, float dtheta,
                  float qw, float qx, float qy, float qz) {
    uint8_t buf[64];
    Frame_t *f = (Frame_t *)buf;
    f->head = FRAME_HEAD;
    f->type = ODOM_IMU;
    f->len  = 7 * sizeof(float);
    memcpy(f->data, (float[]){dx,dy,dtheta,qw,qx,qy,qz}, f->len);
    uint16_t crc = CRC16(buf, sizeof(Frame_t) + f->len);
    memcpy(buf + sizeof(Frame_t) + f->len, &crc, 2);
    sendto(sock, buf, sizeof(Frame_t) + f->len + 2, 0,
           (struct sockaddr *)addr, sizeof(*addr));
}

/* 接收 cmd_vel 帧并解析 */
void UDP_RecvCmd(int sock, float *v, float *omega) {
    uint8_t buf[64];
    int n = recvfrom(sock, buf, sizeof(buf), 0, NULL, NULL);
    Frame_t *f = (Frame_t *)buf;
    if (f->head != FRAME_HEAD || f->type != CMD_VEL) return;
    uint16_t crc = CRC16(buf, n - 2);
    if (memcmp(&crc, buf + n - 2, 2) != 0) return;  // 校验失败
    memcpy(v,     f->data,                sizeof(float));
    memcpy(omega, f->data + sizeof(float), sizeof(float));
}

3   上位机：ROS2 导航栈

3.1   Cartographer SLAM 建图

采用 Google Cartographer 进行二维激光 SLAM。输入为 LiDAR 点云（/scan） 与 odom/IMU（/odom, /imu），输出栅格地图（OccupancyGrid）。 Cartographer 内部使用子图（submap）拼接机制与回环检测（loop closure）， 在长走廊与大房间场景中均能保持地图一致性。

3.2   AMCL 定位

建图完成后，使用 AMCL（Adaptive Monte Carlo Localization）在已知地图上进行粒子滤波定位。 AMCL 根据 LiDAR 观测与地图的匹配程度动态调整粒子数量（100–5000）， 在计算效率与定位精度之间自适应平衡。

3.3   Nav2 路径规划

Nav2 框架提供完整的导航能力：

  • 全局规划器（NavFn）：在全局代价地图上使用 Dijkstra/A* 算法计算从起点到目标点的最优路径
  • 局部规划器（DWB）：Dynamic Window approach 生成候选轨迹，评估碰撞风险与路径跟随误差，输出实时 cmd_vel
  • 恢复行为（Recovery）：遇到卡死时自动执行旋转/后退，尝试脱困

4   定制 PCB 设计

4.1   电源管理

电池输入 7.4V（2S LiPo），经 DC-DC 降压至 5V（给 RPi 供电）， 再由 LDO 稳压至 3.3V（ESP32-S3 逻辑电平）。 5V 与 3.3V 均设有去耦电容与 TVS 管保护。

4.2   电机驱动

采用 RZ7899 双 H 桥驱动芯片，单片驱动两路电机（每路最大 1.5A 连续电流）。 PWM 频率 20 kHz，超出人耳听觉范围以减少噪声。 IN1/IN2 控制正反转，ENA 使能/禁用。

4.3   ESD 防护

USB 接口与 LiDAR 串口引脚处均加置 ESD 保护二极管（TVS 阵列）， 抑制静电放电对芯片的损害。PCB 布局遵循信号完整性原则， 电源走线宽度 ≥ 20 mil，高速信号（SPI、I²C）布线长度匹配。
