\subsection{项目背景与目标}

RoboMaster机甲大师赛要求工程机器人在高速对抗环境下实现自主``取矿--兑换''操作。核心挑战在于：机械臂末端需精准对准兑换站上的4个L形蓝色LED标记（30$\times$30cm正方形四角），完成6-DoF位姿估计后驱动4连杆机械臂执行抓取，整个闭环延迟须低于20ms才能满足实时性要求。

本项目自研了一套\textbf{跨平台C++17视觉伺服引擎框架}（rm-engine），在不依赖GPU加速的Intel NUC边缘设备上，实现了从鱼眼图像采集、传统视觉检测、6-DoF PnP位姿估计、解析逆运动学求解、笛卡尔轨迹插值到CRC16串口闭环反馈的\textbf{全链路自动操控}，端到端延迟仅\textbf{13.5ms}（$\sim$74 FPS）。

端到端流水线分为以下阶段，每帧总延迟仅\textbf{13.5ms}：

\subsubsection{Stage 1--2：鱼眼采集与双色彩空间分割}

采用HSV与灰度\textbf{联合过滤}隔离蓝色LED区域，有效抑制镜面反射与彩色背景干扰：

\begin{equation}
    \mathbf{M} = \mathbf{M}_{\text{HSV}} \cap \mathbf{M}_{\text{gray}}, \quad
    \mathbf{M}_{\text{HSV}} = \mathbf{1}[H \!\in\! [38,179] \wedge S \!\in\! [135,255] \wedge V \!\in\! [100,255]]
\end{equation}

鱼眼相机内参与Brown-Conrady畸变系数（$k_1 = -0.339$的强负值反映桶形畸变特征）：

\begin{equation}
    \mathbf{K} = \begin{bmatrix} 501.36 & 0 & 635.69 \\ 0 & 501.26 & 488.57 \\ 0 & 0 & 1 \end{bmatrix}, \quad
    \mathbf{d} = [-0.339,\; 0.152,\; -1.14\!\times\!10^{-5},\; -3.30\!\times\!10^{-5},\; -0.036]
\end{equation}

\subsubsection{Stage 3--4：边缘增强与多准则轮廓过滤}

\begin{equation}
    \mathbf{E} = \text{Dilate}_{5\times5}\Big(\text{Canny}\big(\text{GaussBlur}_{7\times7,\sigma=5}(\mathbf{M}),\; 50,\; 100\big)\Big)
\end{equation}

通过\textbf{四个几何准则}筛选有效的L形轮廓：

\begin{enumerate}
    \item \textbf{面积约束}：$A_c > 200$ px$^2$（剔除噪声）
    \item \textbf{长宽比}：$0.4 < \text{AR} < 3.5$（剔除细长伪影）
    \item \textbf{多边形顶点数}：$4 \leq N_{\text{poly}} \leq 15$（匹配L形几何，$\varepsilon = 0.02 \times$周长）
    \item \textbf{填充率}：$0.35 \leq A_c/(w \times h) \leq 0.75$（匹配L形面积比）
\end{enumerate}

\begin{lstlisting}[style=cppstyle, caption={HSV分割+Canny边缘+多准则轮廓过滤}]
// 双色彩空间分割
cv::cvtColor(frame, hsv, cv::COLOR_BGR2HSV);
cv::inRange(hsv, cv::Scalar(38,135,100),
            cv::Scalar(179,255,255), mask_hsv);
cv::cvtColor(frame, gray, cv::COLOR_BGR2GRAY);
cv::threshold(gray, mask_gray, 100, 200, cv::THRESH_BINARY);
cv::bitwise_and(mask_hsv, mask_gray, mask);

// 边缘增强: 高斯模糊 -> Canny -> 膨胀
cv::GaussianBlur(mask, mask, cv::Size(7,7), 5);
cv::Canny(mask, edges, 50, 100);
cv::dilate(edges, edges,
    cv::getStructuringElement(cv::MORPH_RECT, cv::Size(5,5)));

// 多准则轮廓过滤
for (const auto& c : contours) {
    double area = cv::contourArea(c);
    if (area < 200) continue;  // 面积约束
    auto rrect = cv::minAreaRect(c);
    float ar = std::max(rrect.size.width, rrect.size.height)
             / std::min(rrect.size.width, rrect.size.height);
    if (ar > 3.5f || ar < 0.4f) continue;  // 长宽比
    double extent = area/(rrect.size.width*rrect.size.height);
    if (extent < 0.35 || extent > 0.75) continue; // 填充率
    valid_contours.push_back(c);
}
\end{lstlisting}

\subsubsection{Stage 5：最小包围圆(MEC)关键点提取}

对通过过滤的$N$个轮廓，枚举所有$\binom{N}{4}$个四元组，选择全局包围圆面积最小的组合，从4个选中圆的边界提取最远点作为关键点：

\begin{equation}
    P_i^{\text{far}} = c_i + r_i \cdot \frac{c_i - C^*}{\|c_i - C^*\|}
\end{equation}

4个关键点按极角$\alpha_i = \text{atan2}(\Delta y, \Delta x)$升序排列，标注为A、B、C、D，确保与3D模型点一一对应后送入PnP求解。

\subsubsection{Stage 6：PnP位姿估计与手眼坐标变换}

使用\texttt{cv::solvePnP}（ITERATIVE模式）在求解过程中内联处理Brown-Conrady畸变校正，恢复6-DoF位姿后，通过\textbf{手眼标定外参偏移}将相机坐标系转换到机械臂基座坐标系：

\begin{equation}
    a_x = t_z - 36, \quad a_y = -t_x - 5, \quad a_z = -t_y + 55 \quad (\text{单位: cm})
\end{equation}

姿态角直接传递：$a_{\text{pitch}} = r_x$，$a_{\text{yaw}} = -r_y$，$a_{\text{roll}} = r_z$。

\subsection{解析逆运动学求解器}

4连杆机械臂（$l_1 = l_2 = 28$cm，$l_3 = 7$cm，$l_4 = 12$cm）的自由度分配为：大臂$\theta_1$（X方向）、小臂$\theta_2$（Y方向）、底座升降（Z轴）、腕部3-DoF（$\psi, \phi, \rho$）。

\subsubsection{求解算法}

\textbf{Step 1：腕部中心解耦}
\begin{equation}
    x_w = a_x - (l_3 + l_4\cos\phi)\cos\psi, \quad
    y_w = a_y - (l_3 + l_4\cos\phi)\sin\psi, \quad
    z_w = a_z - l_4\sin\phi
\end{equation}

\textbf{Step 2：余弦定理求解}——平面到达距离$r = \sqrt{x_w^2 + y_w^2}$，工作空间约束$|l_1 - l_2| \leq r \leq l_1 + l_2$：

\begin{equation}
    \cos\theta_2 = \text{clamp}\!\left(\frac{r^2 - l_1^2 - l_2^2}{2l_1 l_2},\; -1,\; 1\right)
\end{equation}

\begin{equation}
    \theta_1 = \text{atan2}(y_w, x_w) \pm \arccos\!\left(\text{clamp}\!\left(\frac{l_1^2 + r^2 - l_2^2}{2l_1 r},\; -1,\; 1\right)\right)
\end{equation}

\textbf{Step 3：双分支消歧与正运动学验证}——$\theta_1$和$\theta_2$的$\pm$组合产生最多4个候选解，通过关节限位（$\theta_1 \in [\pi/2, \pi]$，$\theta_2 \in [-\pi, 2\pi]$）和正运动学验证（末端误差$< 0.5$cm）筛选，最终调整偏航角$\psi_{\text{final}} = \psi - \theta_1 - \theta_2$。

\textbf{Step 4：工作空间扩展}——当$r > 80$cm时触发底盘前移补偿；若IK仍失败则复位至安全构型。

\subsection{编码器与航迹推算}

通过TIM2/TIM5编码器模式采集轮速，实现航迹推算（Dead Reckoning）：

\begin{equation}
    \Delta s = \frac{\Delta s_L + \Delta s_R}{2}, \quad
    \Delta\theta = \frac{\Delta s_R - \Delta s_L}{D}
\end{equation}

\begin{equation}
    x_{k+1} = x_k + \Delta s \cdot \cos\!\left(\theta_k + \frac{\Delta\theta}{2}\right), \quad
    y_{k+1} = y_k + \Delta s \cdot \sin\!\left(\theta_k + \frac{\Delta\theta}{2}\right)
\end{equation}

其中 $D$ 为两轮轴距，$\Delta s_L$、$\Delta s_R$ 分别为左右轮弧长增量。结合JY901S IMU的航向角进行\textbf{加权融合}修正累积漂移。

\subsection{PID闭环速度控制}

\begin{lstlisting}[style=cstyle, caption={增量式PID速度控制器（50ms采样周期）}]
typedef struct {
    float Kp, Ki, Kd;       // Kp=0.1, Ki=0.1, Kd=0.001
    float error, last_error, prev_error;
    float output;
    float out_max, out_min;
} PID_TypeDef;

float PID_Incremental(PID_TypeDef *pid, float target, float actual) {
    pid->error = target - actual;

    // 增量式PID: delta_u = Kp*(e-e1) + Ki*e + Kd*(e-2e1+e2)
    float delta = pid->Kp * (pid->error - pid->last_error)
                + pid->Ki * pid->error
                + pid->Kd * (pid->error - 2*pid->last_error
                             + pid->prev_error);

    pid->output += delta;

    // 输出限幅，防止PWM溢出
    if (pid->output > pid->out_max) pid->output = pid->out_max;
    if (pid->output < pid->out_min) pid->output = pid->out_min;

    pid->prev_error = pid->last_error;
    pid->last_error = pid->error;

    return pid->output;
}
\end{lstlisting}

// JY901S 数据帧格式: 0x55 + TYPE + 8字节数据 + SUM
void IMU_ParseFrame(uint8_t *buf) {
    if (buf[0] != 0x55) return;

    uint8_t type = buf[1];
    switch (type) {
        case 0x53:  // 欧拉角数据帧
            // Roll/Pitch/Yaw = raw * (180.0 / 32768.0)
            imu_data.roll  = ((int16_t)(buf[3]<<8|buf[2]))
                             / 32768.0f * 180.0f;
            imu_data.pitch = ((int16_t)(buf[5]<<8|buf[4]))
                             / 32768.0f * 180.0f;
            imu_data.yaw   = ((int16_t)(buf[7]<<8|buf[6]))
                             / 32768.0f * 180.0f;
            break;
        case 0x51:  // 加速度数据帧
            imu_data.ax = ((int16_t)(buf[3]<<8|buf[2]))
                          / 32768.0f * 16.0f * 9.8f;
            imu_data.ay = ((int16_t)(buf[5]<<8|buf[4]))
                          / 32768.0f * 16.0f * 9.8f;
            break;
    }
}

    uint8_t frame[14];
    frame[0] = 0xBB;  // 帧头
    // 编码器脉冲数 (左/右)
    frame[1] = (encoder_left >> 8) & 0xFF;
    frame[2] = encoder_left & 0xFF;
    frame[3] = (encoder_right >> 8) & 0xFF;
    frame[4] = encoder_right & 0xFF;

    frame[5] = (yaw_raw >> 8) & 0xFF;
    frame[6] = yaw_raw & 0xFF;
    // 传感器状态 (碰撞+悬崖)

    uint8_t sum = 0;

\subsection{ADC双通道电压电流检测}

原理图参数：电压分压 $R_{16}=33\text{k}\Omega$, $R_{22}=10\text{k}\Omega$；电流检测 $R_{shunt}=0.01\Omega$, 放大倍数$\times$20：

\begin{equation}
    V_{in} = V_{adc} \times \frac{R_{16} + R_{22}}{R_{22}} = V_{adc} \times 4.3
\end{equation}

\begin{equation}
    I = \frac{V_{out} - V_{ref}}{20 \times R_{shunt}} = 5 \times (V_{out} - 1.2)
\end{equation}

\begin{lstlisting}[style=cstyle, caption={ADC双通道序列采集 + 16次滑动平均滤波}]
#define ADC_VREF        3.3f
#define ADC_MAX_COUNTS  4095.0f
#define VOL_RATIO       4.3f      // 电压分压比
#define CUR_VREF        1.2f      // 电流检测参考电压
#define CUR_GAIN        5.0f      // 电流增益 A/V
#define ADC_AVG_N       16        // 平均滤波次数

static void ADC_Read_CurVol_Avg(uint16_t *raw_cur,
                                 uint16_t *raw_vol)
{
    uint32_t sum_cur = 0, sum_vol = 0;
    for (int n = 0; n < ADC_AVG_N; n++) {
        HAL_ADC_Start(&hadc1);
        // Rank1 = CH5 = ADC_CUR (PA5)
        if (HAL_ADC_PollForConversion(&hadc1, 10) == HAL_OK)
            sum_cur += HAL_ADC_GetValue(&hadc1);
        // Rank2 = CH6 = ADC_VOL (PA6)
        if (HAL_ADC_PollForConversion(&hadc1, 10) == HAL_OK)
            sum_vol += HAL_ADC_GetValue(&hadc1);
        HAL_ADC_Stop(&hadc1);
    }
    *raw_cur = (uint16_t)(sum_cur / ADC_AVG_N);
    *raw_vol = (uint16_t)(sum_vol / ADC_AVG_N);
}

static void Measure_Update(void) {
    ADC_Read_CurVol_Avg(&adc_raw_cur, &adc_raw_vol);
    float v_cur = (adc_raw_cur * ADC_VREF) / ADC_MAX_COUNTS;
    float v_vol = (adc_raw_vol * ADC_VREF) / ADC_MAX_COUNTS;
    meas_voltage_V = v_vol * VOL_RATIO;        // 真实电压
    meas_current_A = (v_cur - CUR_VREF) * CUR_GAIN; // 真实电流
    if (meas_current_A < 0.0f) meas_current_A = 0.0f;
}
\end{lstlisting}

\subsection{PI闭环控制器}

PI控制器基于离散化实现，采样周期 $T = 2$ ms（500 Hz）：

\begin{equation}
    V_{PI}(k) = K_p \cdot e(k) + K_i \sum_{n=0}^{k} e(n) \cdot T
\end{equation}

\begin{lstlisting}[style=cstyle, caption={PI控制器离散实现（500Hz采样频率）}]
float Kp = 0.8f;
float T  = 0.002f;           // 采样周期 2ms = 500Hz
float Ki = Kp / T;           // Ki = 400
float Ve, VI = 0.0f, Vpi;

// 在定时器中断中每2ms调用一次
void PI_Control_Update(float Vin, float Vref) {
    Ve = Vin - Vref;          // 误差 = 实际值 - 目标值
    VI += Ki * T * Ve;        // 积分项累加
    // 积分限幅防止windup
    if (VI > PI_MAX) VI = PI_MAX;
    if (VI < PI_MIN) VI = PI_MIN;
    Vpi = Kp * Ve + VI;      // PI输出 = 比例项 + 积分项
}
\end{lstlisting}

int main(void) {
    HAL_Init();
    SystemClock_Config();   // 72MHz HSE + PLL
    MX_GPIO_Init();
    MX_ADC1_Init();
    MX_TIM2_Init();
    MX_TIM3_Init();
    HAL_ADCEx_Calibration_Start(&hadc1); // ADC校准
    OLED_Init();
    Measure_Update();
    OLED_ShowMeas(Motor_Speed, Motor_Dir,
                  meas_voltage_V, meas_current_A);

    while (1) {
        Key_status = Key_Scan();
        if (Key_status != 0) {
            switch (Key_status) {
                case 1: Motor_Stop(); break;      // 停止
                case 2: Motor_Dir = !Motor_Dir;    // 换向
                        Motor_UpdateOutput(); break;
                case 3: Motor_Speed += Motor_Speed_Step; // 加速
                        if (Motor_Speed > 2300) Motor_Speed = 2300;
                        Motor_UpdateOutput(); break;
                case 4: Motor_Speed -= Motor_Speed_Step; // 减速
                        if (Motor_Speed < 100) Motor_Speed = 100;
                        Motor_UpdateOutput(); break;

            OLED_ShowMeas(Motor_Speed, Motor_Dir,
                          meas_voltage_V, meas_current_A);

        static uint32_t meas_tick = 0;
        if (HAL_GetTick() - meas_tick >= 100) {
            meas_tick = HAL_GetTick();
            Measure_Update();
            OLED_ShowMeas(Motor_Speed, Motor_Dir,
                          meas_voltage_V, meas_current_A);

\subsubsection{PCA主成分分析}

\begin{lstlisting}[style=matlabstyle, caption={PCA.m -- 协方差矩阵特征分解降维}]
function [pcaA, V, conVec] = PCA(A, k)
% 输入: A -- 样本矩阵(N x 2576), k -- 目标维度
% 输出: pcaA -- 降维后特征, V -- 主成分向量

[row, col] = size(A);

% Step 1: 去除均值（中心化）
meanA = mean(A);
meanVec = A - meanA;

% Step 2: 计算协方差矩阵 (使用 A'A 替代 AA' 加速)
conVec = meanVec' * meanVec / row;

% Step 3: 计算前k个最大特征值对应的特征向量
[V, D] = eigs(conVec, k);

% Step 4: 特征向量归一化为单位向量
for i = 1:k
    V(:,i) = V(:,i) / norm(V(:,i));
end

% Step 5: 将原始样本投影到k维子空间
pcaA = meanVec * V;
\end{lstlisting}

PCA的数学原理：设样本矩阵 $\mathbf{X} \in \mathbb{R}^{N \times d}$（$d=2576$），中心化后的协方差矩阵：

\begin{equation}
    \mathbf{C} = \frac{1}{N} \bar{\mathbf{X}}^T \bar{\mathbf{X}} \in \mathbb{R}^{d \times d}
\end{equation}

取前$k$个最大特征值对应的特征向量 $\mathbf{V} = [\mathbf{v}_1, \ldots, \mathbf{v}_k]$，降维结果：
\begin{equation}
    \mathbf{Y} = \bar{\mathbf{X}} \cdot \mathbf{V} \in \mathbb{R}^{N \times k}
\end{equation}

function [scaledData, minVals, maxVals] = scaling(data)
    minVals = min(data);
    maxVals = max(data);
    range = maxVals - minVals;
    range(range == 0) = 1;  % 避免除零
    scaledData = 2 * (data - minVals) ./ range - 1;
end

    n_class = length(y_labels);  % 27
    models = cell(n_class, 1);

end

\subsubsection{Bagging概率投票预测}

\begin{lstlisting}[style=matlabstyle, caption={multiSVM.m -- Bagging后验概率投票}]
function [pred, acc] = multiSVM(X_test, testlabel, y_train, models)
    y_labels = unique(y_train);
    n_class = 27;
    test_num = size(X_test, 1);
    y_predict = zeros(test_num, 1);

    for i = 1:test_num
        bagging = zeros(n_class, 1);
        for j = 1:n_class
            model = models{j};
            [~, rat] = predict(model, X_test(i,:));
            bagging(j) = bagging(j) + rat(2);
        end
        [~, maxp] = max(bagging);
        y_predict(i) = y_labels(maxp);
    end

    right = sum(y_predict == testlabel);
    acc = right / test_num;
    pred = y_predict;
    fprintf('识别准确率: %.2f%%\n', acc * 100);
end
\end{lstlisting}

\begin{lstlisting}[style=vhdlstyle, caption={四位同步可逆计数器完整实现（test31.vhd）}]
library IEEE;
use IEEE.STD_LOGIC_1164.ALL;
use IEEE.STD_LOGIC_UNSIGNED.ALL;

entity test31 is
    Port (
        clk   : in  STD_LOGIC;
        clr   : in  STD_LOGIC;     -- 异步清零 (高有效)
        s     : in  STD_LOGIC;     -- 并行加载使能
        en    : in  STD_LOGIC;     -- 计数使能
        updn  : in  STD_LOGIC;     -- 1=上计数, 0=下计数
        d     : in  STD_LOGIC_VECTOR(3 downto 0);
        co    : out STD_LOGIC;     -- 进位/借位输出
        q     : buffer STD_LOGIC_VECTOR(3 downto 0)
    );
end test31;

architecture Behavioral of test31 is
begin
    process(clk, clr)
    begin
        if clr = '1' then
            q <= "0000";           -- 异步清零
        elsif rising_edge(clk) then
            if s = '1' then
                q <= d;            -- 并行加载
            elsif en = '1' then
                if updn = '1' then
                    q <= q + 1;    -- 上升计数
                else
                    q <= q - 1;    -- 下降计数
                end if;
            end if;
        end if;
    end process;

    -- 进位输出: 上计数到1111或下计数到0000时置1
    co <= '1' when (updn='1' and q="1111" and en='1')
              or   (updn='0' and q="0000" and en='1')
          else '0';
end Behavioral;
\end{lstlisting}

entity shift_4 is
    Port (
        clk         : in  STD_LOGIC;
        reset       : in  STD_LOGIC;
        shift_left  : in  STD_LOGIC;
        shift_right : in  STD_LOGIC;
        data_in     : in  STD_LOGIC_VECTOR(3 downto 0);
        data_out    : out STD_LOGIC_VECTOR(3 downto 0)
    );
end shift_4;

architecture Behavioral of shift_4 is
    signal reg : STD_LOGIC_VECTOR(3 downto 0);
begin
    process(clk, reset)
    begin
        if reset = '1' then
            reg <= "0000";
        elsif rising_edge(clk) then
            if shift_left = '1' then
                -- 循环左移: 最高位移到最低位
                reg <= reg(2 downto 0) & reg(3);
            elsif shift_right = '1' then
                -- 循环右移: 最低位移到最高位
                reg <= reg(0) & reg(3 downto 1);
            else
                reg <= data_in;  -- 并行加载
            end if;
        end if;
    end process;
    data_out <= reg;
end Behavioral;
\end{lstlisting}

                end if;
            end if;
        end if;
    end process;

\subsubsection{ADS1292R数据读取}

SPI读取ADS1292R的原始数据帧格式为：3字节状态 + 3字节通道1数据 + 3字节通道2数据，共9字节。24位有符号数需要扩展为32位：

\begin{lstlisting}[style=cstyle, caption={ADS1292R SPI数据读取与24位符号扩展}]
// 发送读取数据命令
void ADS1292_ReadData(int32_t *ch1_data) {
    uint8_t buff[10];
    // 发送 RDATA 命令 (0x12)
    ADS1292_SendCommand(CMD_RDATA);

    // 读取 9 字节: 3字节状态 + 3字节CH1 + 3字节CH2
    HAL_SPI_Receive(&hspi1, &buff[1], 9, 100);

    // 提取通道1的24位数据 (高字节在前)
    int32_t raw = ((int32_t)buff[4] << 16)
               | ((int32_t)buff[5] << 8)
               | ((int32_t)buff[6]);

    // 24位有符号数扩展为32位 (补码符号位扩展)
    if (raw & 0x800000) {      // bit23 = 1 表示负数
        raw |= 0xFF000000;     // 高8位补1
    }

    *ch1_data = raw;
}
\end{lstlisting}

\noindent 读取得到的原始数据需要按照芯片手册解码，乘以增益系数恢复为实际电压值（单位mV）。

    \item \textbf{Pan-Tompkins R波检测算法}：
    \begin{enumerate}[label=(\alph*)]

        \item \textbf{平方运算}：增强大斜率区域，抑制小信号

    \item \textbf{LCD显示}：实时绘制滤波后心电波形、频谱图、心率数值
\end{enumerate}

\subsection{Pan-Tompkins算法数学表达}

\textbf{五点差分}（突出QRS斜率）：
\begin{equation}
    y[n] = \frac{1}{8}\left(2x[n] + x[n\!-\!1] - x[n\!-\!3] - 2x[n\!-\!4]\right)
\end{equation}

\textbf{平方增强}：
\begin{equation}
    z[n] = y[n]^2
\end{equation}

\textbf{滑动窗口积分}（$N \approx 0.15 \times f_s$）：
\begin{equation}
    w[n] = \frac{1}{N}\sum_{k=0}^{N-1} z[n-k]
\end{equation}

\textbf{心率计算}：
\begin{equation}
    \text{BPM} = \frac{60 \times f_s}{\overline{RR}}
\end{equation}

其中 $\overline{RR}$ 为平均R-R间期采样点数，$f_s$ 为采样频率。

\subsection{自适应阈值策略}

Pan-Tompkins算法的关键在于\textbf{自适应双阈值}机制：

\begin{lstlisting}[style=matlabstyle, caption={Pan-Tompkins自适应阈值R波检测}]
% 初始化阈值
SPKI = max(signal(1:2*fs));   % 信号峰值初始估计
NPKI = mean(signal(1:2*fs));  % 噪声峰值初始估计
THR1 = NPKI + 0.25*(SPKI - NPKI); % 检测阈值
THR2 = 0.5 * THR1;            % 回搜阈值

R_peaks = [];
for i = 2:length(signal)-1
    if signal(i) > signal(i-1) && signal(i) > signal(i+1)
        if signal(i) > THR1
            % 检测到R波
            R_peaks = [R_peaks, i];
            % 更新信号峰值估计
            SPKI = 0.125*signal(i) + 0.875*SPKI;
        else
            % 更新噪声峰值估计
            NPKI = 0.125*signal(i) + 0.875*NPKI;
        end
        % 动态更新阈值
        THR1 = NPKI + 0.25*(SPKI - NPKI);
        THR2 = 0.5 * THR1;
    end
end

% 计算心率
RR_intervals = diff(R_peaks);
BPM = 60 * fs / mean(RR_intervals);
fprintf('心率: %.1f BPM\n', BPM);
\end{lstlisting}

\subsection{3-DOF逆运动学解算}

台灯采用3自由度串联连杆机构，连杆长度$L_1 = L_2 = 145\text{mm}$。系统采用几何法进行逆运动学解算，将目标位置表示为等腰三角形参数：

\begin{itemize}
    \item $b$：等腰三角形底边长（米），范围$0 \sim 0.28$m
    \item $\theta_0$：底边角度（台灯朝向），范围$0° \sim 180°$
    \item $\beta$：灯头俯仰角，范围$-30° \sim 60°$
\end{itemize}

逆解算公式：
\begin{equation}
    \theta_1 = \arcsin\left(\frac{b}{2a}\right), \quad
    \alpha_1 = \theta_0, \quad
    \alpha_2 = 90° - \theta_1, \quad
    \alpha_3 = 180° - 2\theta_1 + \beta
\end{equation}

其中$a = 0.145$m为连杆长度。舵机编码与角度的转换关系为：
\begin{equation}
    \text{encoder} = \text{zero\_pos} + \text{direction} \times \left(\text{angle} \times \frac{410}{90}\right)
\end{equation}

\subsection{视觉系统——多色HSV识别}

\begin{lstlisting}[style=pythonstyle, caption={OpenCV多色HSV空间色块检测}]
import cv2
import numpy as np

# 四种颜色HSV阈值范围
color_ranges = {
    'red':   ([0, 100, 100],   [10, 255, 255]),
    'green': ([35, 100, 100],  [85, 255, 255]),
    'blue':  ([100, 100, 100], [130, 255, 255]),
    'yellow':([20, 100, 100],  [35, 255, 255]),
}

def detect_blocks(frame):
    hsv = cv2.cvtColor(frame, cv2.COLOR_BGR2HSV)
    results = []

    for color_name, (lower, upper) in color_ranges.items():
        mask = cv2.inRange(hsv, np.array(lower),
                           np.array(upper))
        # 形态学处理: 开运算去噪 + 闭运算填充
        kernel = cv2.getStructuringElement(
            cv2.MORPH_ELLIPSE, (5, 5))
        mask = cv2.morphologyEx(mask, cv2.MORPH_OPEN, kernel)
        mask = cv2.morphologyEx(mask, cv2.MORPH_CLOSE, kernel)

        # 轮廓检测
        contours, _ = cv2.findContours(
            mask, cv2.RETR_EXTERNAL,
            cv2.CHAIN_APPROX_SIMPLE)

        for cnt in contours:
            area = cv2.contourArea(cnt)
            if area > 500:  # 最小面积过滤
                x, y, w, h = cv2.boundingRect(cnt)
                cx, cy = x + w//2, y + h//2
                results.append({
                    'color': color_name,
                    'center': (cx, cy),
                    'area': area,
                    'bbox': (x, y, w, h)
                })
    return results
\end{lstlisting}

\subsection{视觉伺服闭环控制}

将色块中心偏差作为PID输入，控制底盘运动趋近目标：

\begin{lstlisting}[style=pythonstyle, caption={视觉伺服PID闭环底盘控制}]
class VisualServoPID:
    def __init__(self):
        self.Kp_x = 0.003   # 水平偏差比例增益
        self.Kp_d = 0.005   # 距离比例增益
        self.target_cx = 320 # 图像中心x (640/2)
        self.target_area = 5000 # 目标面积 (对应期望距离)

    def compute(self, cx, area):
        # 水平偏差 -> 旋转速度
        error_x = cx - self.target_cx
        omega = -self.Kp_x * error_x

        # 面积偏差 -> 前进速度
        error_d = self.target_area - area
        velocity = self.Kp_d * error_d

        return velocity, omega
\end{lstlisting}

\begin{itemize}

为从稀疏点云中识别圆柱形障碍物，采用类RANSAC（Random Sample Consensus）方法：

\begin{enumerate}
    \item 将360个极坐标障碍物点转换为笛卡尔坐标
    \item 对每个障碍物点向外延伸圆半径长度，生成假定圆心候选点
    \item 统计各候选圆心附近的内点（inlier）数量
    \item 内点数最多的候选即为检测到的圆柱中心
\end{enumerate}

\subsection{研究背景与目标}

电磁发射技术（Electromagnetic Launch, EML）利用安培力取代传统化学发射，具有可控性强、能量转换效率可优化等优势。项目以多级电磁线圈炮（Coilgun）为研究对象，核心目标包括：

\begin{enumerate}

\end{enumerate}

\subsection{系统总体设计——模块化架构}

整个电磁炮系统采用模块化设计思路，将复杂系统拆分为三大功能模块：

\begin{itemize}
    \item \textbf{储能模块}：多组铝电解电容并联，提供瞬时大电流放电能力；充电回路通过限流电阻控制充电速率，防护电路包含放电保护与过压保护
    \item \textbf{触发模块}：IGBT功率管作为主开关器件，通过驱动电路将STM32的低压控制信号转换为IGBT栅极驱动信号；每级线圈对应独立的IGBT触发通道
    \item \textbf{光电检测模块}：弹丸经过各级线圈时触发光电传感器，生成中断信号反馈STM32，实现闭环级联触发
\end{itemize}

\begin{iterbox}[核心设计决策]
\textbf{为何选择IGBT而非MOSFET？}——电磁炮放电瞬间电流可达数十安培，电压可达数百伏。IGBT在高压大电流工况下导通压降低、开关损耗可控，比MOSFET更适合脉冲功率应用。但IGBT关断速度相对较慢（$\mu$s级拖尾电流），需要通过实验精确表征关断特性以优化时序。
\end{iterbox}

\subsection{RLC放电理论模型与实验验证}

单级线圈放电过程可等效为RLC串联二阶电路。电容初始电压$V_0$，放电方程为：

\[
L\frac{d^2q}{dt^2} + R\frac{dq}{dt} + \frac{q}{C} = 0
\]

当$R^2 < 4L/C$（欠阻尼条件）时，放电电流为：

\[
i(t) = \frac{V_0}{\omega_d L} e^{-\alpha t} \sin(\omega_d t), \quad \alpha = \frac{R}{2L}, \quad \omega_d = \sqrt{\frac{1}{LC} - \alpha^2}
\]

实验中100$\mu$s/div时基下观察到的衰减振荡波形与上述欠阻尼解对应，电流峰值在第一个半周期达到最大，随后指数衰减，验证了理论模型的正确性。

\subsection{问题引入：单次敲击的时域瞬态分析}

研究最初的兴趣源于最直观的物理好奇：当一个圆形鼓膜受到一次短促敲击时，其膜面位移如何随时间演化？

将实际羊皮鼓抽象为理想二维圆形张紧薄膜，设膜面密度$\rho_s$（kg/m$^2$），张力$T$（N/m），膜上横波传播速度$c = \sqrt{T/\rho_s}$。膜面任意点$(r,\theta)$的位移$u(r,\theta,t)$满足二维波动方程：

\[
\frac{\partial^2 u}{\partial t^2} = c^2 \nabla^2 u
\]

边界条件为固定边界（Dirichlet条件）：$u(a, \theta, t) = 0$。

\begin{figure}[H]
    \centering
    \includegraphics[width=0.5\textwidth]{extracted_images/mathphys_001.png}
    \caption{圆形薄膜COMSOL建模几何及坐标设定}
\end{figure}

\subsubsection{分离变量法求解本征模态}

采用分离变量法$u = R(r)\Theta(\theta)T(t)$，径向部分满足$m$阶Bessel方程，由于膜心$r=0$处位移有限，舍弃第二类Bessel函数$Y_m$，仅保留第一类Bessel函数$J_m$：

\[
R_{mn}(r) = J_m\!\left(\frac{\alpha_{mn}}{a} r\right)
\]

其中$\alpha_{mn}$为$J_m$的第$n$个正零点。空间本征模态为：

\[
\Phi_{mn}(r,\theta) = J_m\!\left(\frac{\alpha_{mn}}{a} r\right)
\begin{cases}
\cos(m\theta) \\
\sin(m\theta)
\end{cases}
\]

对应本征频率为：

\[
f_{mn} = \frac{1}{2\pi} \sqrt{\frac{T}{\rho_s}} \cdot \frac{\alpha_{mn}}{a}
\]

\begin{enumerate}
    \item \textbf{多模态混叠}：一次敲击同时激发大量模态，时域信号是多频叠加
    \item \textbf{瞬态衰减快}：高阶模态在极短时间窗口内衰减殆尽
    \item \textbf{位移幅值小}：毫米甚至亚毫米级位移，普通设备难以精确采集
\end{enumerate}

\subsection{研究路线关键迭代：从时域到频域}

\begin{iterbox}[方法论迭代]
\textbf{核心转折}：上述三大困难表明——时域瞬态敲击几乎不可能"看清"单一模态。要验证Bessel模态形状，\textbf{必须通过稳态共振把单一模态"筛选"出来}，即从"时域瞬态"迭代为"频域稳态扫频响应"。这一转折是整个研究的方法论核心。
\end{iterbox}

\subsubsection{频域稳态理论：Helmholtz方程与共振筛选}

取正弦稳态激励，波动方程转化为非齐次Helmholtz方程。利用本征函数展开，每阶模态的频率响应为：

\[
c_{mn}(\omega) = \frac{F_{mn}}{(\omega_{mn}^2 - \omega^2) + 2j\zeta_{mn}\omega_{mn}\omega}
\]

当驱动频率$\omega \to \omega_{mn}$时，分母趋近最小，该模态被显著放大，其他模态贡献弱——这就是\textbf{共振筛选机理}，也是Chladni盐粒法可行的根本物理原因。

\subsection{核心验证判据：频率比定律}

对同一张鼓膜，材料参数$T$、$\rho_s$和半径$a$均为常数。任意两模态频率之比仅由Bessel零点比决定：

\[
\frac{f_{mn}}{f_{pq}} = \frac{\alpha_{mn}}{\alpha_{pq}}
\]

这一定律的重要意义在于：\textbf{无需精确测量难以获取的膜张力或面密度}，只需测量两个共振频率即可与纯数学的Bessel零点比对照。

\subsection{实验不足与反思}

\begin{enumerate}

    \item \textbf{真实阻尼}：空气辐射、膜材料内阻尼、边界摩擦导致高阶模态共振峰被"压低"
    \item \textbf{显影介质局限}：盐粒粒径不均匀、摩擦力影响迁移效率
    \item \textbf{环境扰动}：温湿度变化导致膜张力漂移，影响实验重复性
\end{enumerate}

\begin{resultbox}[核心结论]

    \item 轴对称激励仅能有效耦合$m=0$模态；非轴对称模态需改进激励方式
\end{itemize}
\end{resultbox}

\subsection{研究背景：隧道洪涝的极端救援需求}

隧道洪灾（如郑州京广隧道2021、广州白云隧道2024）具有空间封闭、水位上涨迅速、能见度极低（0.3--1 m）等极端特征。远程操控救灾机器人是替代人类进入高危水域的关键手段，但操作者面临严峻的信息获取与认知负荷挑战：

\begin{itemize}
    \item 水体混有泥沙油污，摄像头画面高度退化
    \item 单一视角深度线索不足，无法准确判断距离
    \item 多任务并行（导航+搜索+物资投放）导致认知过载
\end{itemize}

\subsection{研究设计：用户需求分析与实验框架}

\subsubsection{双角色需求挖掘}

通过访谈法与焦点小组法，系统梳理两类关键角色的需求：

\textbf{操作者需求}：(1) 环境感知——准确感知障碍物分布与水流状态；(2) 操控精度——在浑浊水体中精确控制运动路径；(3) 认知负荷管理——避免信息过载，关键信息突出显示；(4) 任务协作——与后方指挥中心高效沟通。

\textbf{被困人员需求}：(1) 心理安全——能识别接近物体为救援设备；(2) 物理安全——机器人不造成二次伤害；(3) 互动沟通——传达自身状况。

\subsubsection{界面复杂度渐进设计}

基于需求分析，设计三级复杂度界面（非简单二元对立，而是逐步叠加信息要素）：

\begin{enumerate}
    \item \textbf{基础模拟界面}：单一主摄像头画面，暗光低可视度，无辅助信息——模拟最原始救援状态
    \item \textbf{信息叠加界面}（中等复杂度）：新增二维小地图、风险区域标注、AI语音提示、机器人姿态自稳定
    \item \textbf{高信息密度界面}（高复杂度）：第三人称视角(3PP)、声呐探测、多源信息融合、VR沉浸式操控
\end{enumerate}

\subsubsection{拟人化程度设计}

\begin{itemize}
    \item \textbf{功能导向型}（低拟人化）：流线型机身，高对比度荧光配色，通过指示灯传递状态
    \item \textbf{拟人化叠加型}（高拟人化）：接入DeepSeek大模型语音反馈，姿态/动作表达意图，VR动作映射
\end{itemize}

\subsection{Unity仿真实验}

\subsubsection{实验设计}

任务场景：Unity构建200 m虚拟隧道，水下能见度0.5--2 m渐变，10+辆被淹没车辆随机分布，被试需完成导航$\to$搜索$\to$物资投放。

\subsubsection{因变量测量}

\begin{itemize}
    \item \textbf{任务绩效}：完成时间（系统自动记录）、投放错误次数
    \item \textbf{认知负荷}：参考NASA-TLX的0--10分主观评分（脑力负荷、努力程度、时间压力、头晕程度等）
    \item \textbf{主观体验}：SUS系统易用性量表、TAM技术接受模型、半结构化访谈
\end{itemize}

\subsection{核心实验发现}

\subsubsection{发现1：机器人稳态是效率的"基座"}

任务完成时间分布在34--110 s区间。\textbf{机器人运动稳定性}决定效率下限（平均缩短20--25 s），视觉质量+认知外化工具（小地图、AI提示）决定效率上限。当系统行为不可预测时，\textbf{任何界面优化都会失效}。

\subsubsection{发现2：AI辅助存在"情境适配区间"}

\begin{itemize}
    \item 轻度负荷场景：语音提示即可有效接收
    \item 中等负荷场景：语音+文字提示有效降低努力与时间压力评分（降1--2分）
    \item 极高负荷场景：纯文字信息处理困难，被试反映"来不及看"
\end{itemize}

\subsubsection{发现3：小地图作为"外部化认知支架"}

引入小地图后NASA-TLX综合评分平均下降1--2分。机制：将"大脑内部的空间记忆与推理"转变为"外部视觉的直接检索"。但在任务简单+机器人稳态条件下，小地图的固定存在反而成为干扰。

\subsubsection{发现4：第三人称视角显著降低晕动症}

引入3PP后平均头晕评分从7.3分降至3.6分（降低51\%）。机制：将视觉输入识别为"他者运动"而非"自我运动"，避免与静止身体前庭信号的冲突。

\subsubsection{发现5：拟人化呈"倒U型"效应}

\begin{itemize}
    \item 低拟人化（键鼠）：逻辑清晰、错误率可控，但直觉性不足
    \item 中等拟人化（VR）：在直觉性、控制精度与生理舒适性间取得最佳平衡
    \item 高拟人化（1PP强沉浸）：增强沉浸感但引发视觉--前庭冲突，认知负荷上升
\end{itemize}

\subsection{辅助研究：博物馆展签信息分层的人因分析}

除主体实验外，项目还对\textbf{博物馆展签信息设计}进行了人因分析，前往湖北省博物馆实地调研：

\begin{itemize}
    \item 现场观察20位随机观众：信息过少/过多的展签停留$<$10 s，信息适中展签停留$>$30 s
    \item 问卷调查28份：100\%选择"展品名称"为首要关注，76\%期望字数$<$150字
    \item 最受欢迎展签类型（69.23\%）：信息适中、结构清晰型（C类）
\end{itemize}

\textbf{结论}：有效展签需具备三个特点——具象清晰的描述、适中简洁的篇幅、便于扫读的结构。

\subsection{设计优化建议}

\begin{resultbox}[核心结论]
\begin{itemize}
    \item \textbf{稳态优先}：机器人运动稳定性是所有交互优化的"生效前提"
    \item \textbf{分层自适应}：视觉信息根据环境照度与任务类型动态切换视角策略
    \item \textbf{界面复杂度存在最优区间}：并非"越简单越好"，适中复杂度+清晰结构最优
    \item \textbf{拟人化的核心是映射清晰度}：映射的清晰度应优先于形态的拟真度
    \item \textbf{AI作为"认知安全网"}：在极端场景下提供预设方案甚至一键接管选项
\end{itemize}
\end{resultbox}

\subsection{斯特林循环热力学基础}

斯特林循环由四个基本过程组成，是一种外燃式闭式循环：

\begin{enumerate}
    \item \textbf{等温压缩}（1$\to$2）：气体在低温侧被等温压缩，对外放热
    \[
    Q_{12} = W_{12} = -mR_gT_C \cdot \ln\frac{V_{max}}{V_{min}} = -0.4319\;\text{J}
    \]
    \item \textbf{等容吸热}（2$\to$3）：气体通过回热器从低温侧流向高温侧，等容吸热
    \[
    Q_{23} = mc_V(T_E - T_C) = 0.5968\;\text{J}, \quad W_{23} = 0
    \]
    \item \textbf{等温膨胀}（3$\to$4）：气体在高温侧等温膨胀对外做功
    \[
    Q_{34} = W_{34} = mR_gT_E \cdot \ln\frac{V_{max}}{V_{min}} = 0.7199\;\text{J}
    \]
    \item \textbf{等容放热}（4$\to$1）：气体通过回热器返回低温侧，等容放热
    \[
    Q_{41} = -mc_V(T_E - T_C) = -0.5968\;\text{J}, \quad W_{41} = 0
    \]
\end{enumerate}

单次循环理论做功：

\[
W_i = W_{34} + W_{12} = mR_g(T_E - T_C) \cdot \ln\frac{V_{max}}{V_{min}} = 0.2880\;\text{J}
\]

在回热器效率$\eta_R = 1$的理想条件下，热效率达到卡诺效率：

\[
\eta = \frac{W_i}{Q_{in}} = \frac{T_E - T_C}{T_E} = 40.0\%
\]

\subsection{史密特理论建模}

项目采用\textbf{史密特（Schmidt）理论}对Gamma型发动机进行准稳态热力学分析。核心假设包括：等温压缩/膨胀、理想气体、完全再生热交换、正弦容积变化。

\subsubsection{容积与压力的解析表达}

以排气器活塞上止点为曲柄角$\theta = 0^\circ$，膨胀空间与压缩空间瞬时容积为：

\[
V_E = \frac{V_{SE}}{2}(1 - \cos\theta), \quad V_C = \frac{V_{SE}}{2}(1 + \cos\theta) + \frac{V_{SC}}{2}[1 - \cos(\theta - \alpha)]
\]

瞬时压力由压力方程给出：

\[
p = \frac{p_{mean}\sqrt{1 - \delta^2}}{1 - \delta\cos(\theta - \phi)}
\]

其中$\delta = B/S$，$B$、$S$为由温度比$\tau = T_C/T_E$、行程容积比$\kappa$和死容积比$\chi$确定的参数。

\subsection{研究背景与课题分析}

当部分装满液体的容器进行垂直振荡并在底部注入空气时，液体可以稳定地``悬浮''在空气层之上。这一反直觉的现象涉及流体动力学与多相流稳定性的核心问题。2020年Apffel等人在\textit{Nature}上发表的研究表明，通过精确校准的垂直振荡，不仅可以使液体层悬浮，甚至可以在悬浮液体的下界面实现``倒置浮力''——物体能够在液体下方倒挂漂浮。

课题要求系统研究以下因素对流体悬浮现象的影响：

\begin{itemize}
    \item 容器的形状、大小等几何参数
    \item 液体的动力粘度和种类（硅油、甘油等）
    \item 振动的频率、幅度和波形
    \item 注入空气的压力、流速和注射位置
    \item 液体的初始填充量
\end{itemize}

\subsection{理论分析}

\subsubsection{Rayleigh--Taylor不稳定性与静态分析}

当密度较大的流体位于密度较小的流体上方时，界面会发生Rayleigh--Taylor（RT）不稳定性。在静态情况下，液体位于气体下方时的正常界面满足重力--毛细管色散关系：

\begin{equation}
\omega_0^2(k) = gk + \frac{\gamma k^3}{\rho_l}
\end{equation}

其中$\omega_0$为自然振荡频率，$k$为波数，$\gamma$为表面张力系数，$\rho_l$为液体密度。重力和毛细力均为正值，界面稳定。

当液体位于气体上方（倒置构型）时：

\begin{equation}
\omega_0^2(k) = -gk + \frac{\gamma k^3}{\rho_l}
\end{equation}

对于小波数$k$（长波），重力项$-gk$占主导，使$\omega_0^2(k)<0$，界面扰动随时间呈指数增长，液体最终坍塌。最不稳定的波数$k^*$由$\omega_0^2(k)$取极小值确定，对于大波矢量（短波），毛细力占主导并具有稳定作用。

\subsubsection{Kapitza效应与动态稳定化}

实验体系的核心思想类比\textbf{卡皮察倒立摆}（Stephenson--Kapitza pendulum）：通过高频垂直振荡在时间平均意义上产生``振动力''，从而稳定原本不稳定的倒置构型。

假设液体受到垂直振荡$z_b(t) = A\cos(\omega t)$，液体的等效加速度为：

\begin{equation}
g_{\mathrm{eff}}(t) = g - A\omega^2\cos(\omega t)
\end{equation}

将线性化的Bernoulli方程应用于界面变形$\zeta$，得到受迫振子方程：

\begin{equation}
\ddot{\zeta} + \omega_0^2(k)\,\zeta = -kA_1\omega^2\cos(\omega t)\,\zeta
\end{equation}

其中$A_1$为液层的振荡幅度。按照Kapitza分析方法，将$\zeta$分解为慢分量$\zeta_s$与快分量$\zeta_f$：

\begin{equation}
\zeta = \zeta_s + \zeta_f
\end{equation}

对方程取时间平均（利用$\langle\cos^2(\omega t)\rangle = 1/2$），得到慢分量的有效运动方程：

\begin{equation}
\ddot{\zeta}_s + \omega_0^2(k)\,\zeta_s = -\frac{k^2A_1^2}{2}\omega^2\,\zeta_s
\end{equation}

右端项$-\frac{k^2A_1^2}{2}\omega^2\zeta_s$类比弹簧恢复力，是振动引起的\textbf{平均稳定化效应}。当$\zeta_s$偏离平衡位置时，该项提供抑制力，阻止界面变形进一步发展。

\textbf{稳定化条件推导}：为使倒置界面稳定，振动稳定化项必须克服重力不稳定项：

\begin{equation}
\frac{k^2A_1^2}{2}\omega^2 > |\omega_0^2(k)|
\end{equation}

对于不稳定模式$\omega_0^2(k)<0$，且小$k$时$|\omega_0^2(k)|\approx gk$，简化得：

\begin{equation}
k > \frac{2g}{A_1^2\omega^2}
\end{equation}

对于横截面积尺度为$L$的容器，可激发的最小波数为$k_{\min} = \pi/L$，因此必须满足：

\begin{equation}
\boxed{A_1\omega > \sqrt{\frac{2gL}{\pi}}}
\end{equation}

该条件给出了实现悬浮所需的最小激励速度，且与容器横截面长度$L$的平方根成正比。

\subsubsection{气泡临界下沉深度}

在振荡液柱中，气泡受到加速度调制$g_{\mathrm{eff}}$和压强调制$P(t)$的联合作用。对气泡体积进行泰勒展开分析表明，除了常规的阿基米德浮力外，还存在额外的\textbf{Bjerknes力}——一种由压力梯度振荡引起的时间平均力。

气泡的平衡方程考虑以下力的贡献：

\begin{itemize}
    \item 重力：$F_g = -\rho_b V g$
    \item 浮力：$F_b = \rho_l V g$
    \item 粘性阻力：$F_d = -\frac{1}{2}\pi r^2 \rho_l C_d u^2$，其中$C_d$为阻力系数
    \item Basset力：$F_B = K_B \cdot 6\pi r \mu \int_0^t \frac{\dot{u}(\tau)}{\sqrt{\pi\nu(t-\tau)}}\,\mathrm{d}\tau$，其中$K_B$为Basset力经验系数
    \item 附加质量力：$F_m = K_m \cdot \frac{1}{2}\rho_l V \dot{u}$，其中$K_m$为附加质量力经验系数
\end{itemize}

实验表明气泡的临界下沉深度$d^*$与振幅$A$的关系满足：

\begin{equation}
d^* = \frac{2gP_0}{\rho_l A^2 \omega^4} \cdot C
\end{equation}

其中特征体积$C$与气泡半径的平方和体积调制有关。高粘度液体不容易出现法拉第不稳定性，且Basset力会减弱浮力，有利于气泡保持稳定。

\subsubsection{负重力效应与倒置浮力}

在悬浮液层的下界面，振动不仅抑制了RT不稳定性，还产生了``负重力''效应。浮体在振动液体界面上的动态稳定条件为：

\begin{equation}
\omega_{\mathrm{buoy}}^2(1+\alpha) > 0, \quad \alpha = \frac{\omega_{\mathrm{buoy}}^2}{2}\left(\frac{A\omega}{g}\right)^2
\end{equation}

当振动足够强时，动态稳定化项$\alpha$使原本为负的有效势能变为极小值，物体可以在悬浮液体的下界面``倒置漂浮''。

\subsubsection{实验器材与方法}

实验器材包括：有机玻璃容器、克拉尼图形演示仪振动台、波形发生器、功率放大器、硅油/甘油（100\,cst）以及注射器。判定标准为：

    \item \textbf{坍塌}：气体层从一侧开始坍塌，气泡上浮

\subsubsection{频率影响实验}

控制注入悬浮气体体积约1\,mL，改变振动频率范围70--250\,Hz。实验发现存在一个临界频率，低于该频率时液层无法保持悬浮，高于该频率时稳定性随频率增大而增强。在更高频率区间，当达到法拉第不稳定性阈值时，界面会出现反对称模式——空气层体积不再恒定，上层液体在摇摆中最终坍塌。
